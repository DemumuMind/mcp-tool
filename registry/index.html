How to use this page

Registry data is support context, not the first stop

  • Use browse and tool dossiers when you need compatibility, adoption, ranking, and alternatives.
  • Use this route when you need the registry contract, cleanup status, and listing workflow rules behind those public surfaces.
  • Use methodology and stats if you want marketplace-wide context before you touch the registry layer directly.

What registry-listed means

  • The tool has been reviewed and added to registry.json.
  • Its name, description, tags, and install contract are curated.
  • Clients such as the public tools directory can discover it reliably.

How to add a tool

  1. Push the repo into the DemumuMind organization.
  2. Open a PR against mcp-tool-registry.
  3. Include the canonical metadata: ID, name, description, repo URL, install object, and tags.
  4. After merge, the next sync makes the listing visible here.
