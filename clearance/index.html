Where it fits

Use clearance after tool/category/platform discovery, when you need brand and package confidence before launch or rename decisions.

How to use this page

Validate naming risk after you shortlist the product

  • Use browse and dossiers first to understand the product, market lane, and adoption context.
  • Use this page when you need namespace availability before shipping a new tool, package, or rename.
  • Use methodology and stats if you need the broader marketplace framing around naming and launch discipline.

Scope

What it checks

Included

  • Namespace availability across npm, PyPI, GitHub, crates.io, Docker Hub, Hugging Face, and domains.
  • Phonetic, homoglyph, and fuzzy variants to surface near-conflicts.
  • Deterministic GREEN / YELLOW / RED scoring with explainable reasoning.
  • Reservation links so you can claim handles immediately.
  • Local execution with no API keys required for the core scan.

Not included

  • Trademark database search across USPTO, EUIPO, or WIPO.
  • Legal advice or legal opinion.
  • Automatic namespace registration or claims.
  • Replacement for trademark counsel.

Quickstart

Run a scan in seconds

One-off execution:

npx @demumumind/clearance-opinion-engine check my-tool-name

Install for repeated use:

npm install -g @demumumind/clearance-opinion-engine
coe check my-tool-name --radar --suggest

Decision model

How the tiers work

Green

All namespaces available. No direct conflicts detected. Safe to reserve now.

Yellow

Some concerns surfaced. Review near-conflicts before committing the brand.

Red

Direct conflicts detected. Rename before the cost compounds.
