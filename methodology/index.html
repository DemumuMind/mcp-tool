Methodology

How ranking and verified status actually work.

DemumuMind MCP uses explicit factors for ranking, verified status, and catalog eligibility. This page explains those rules and where the model stops being a guarantee.

Ranking

Weighted quality score

Ranking prioritizes documentation, adoption path, freshness, popularity, trust signals, and media completeness. Deprecated tools are strongly penalized.

  • Docs: homepage, docs, and a clear summary lift the score.
  • Adoption: install guidance and use-case framing matter.
  • Freshness: recently updated tools rise above stale ones.
  • Trust: release and proof signals help, but do not replace docs.

Verified

Verified is automatic, not a paid badge

Verified is automatic in v1. It reflects a score threshold plus minimum listing quality, not sponsorship and not a hand-managed certification program.

Eligibility

Primary catalog rules

  • Every primary listing needs a clear summary of the MCP surface.
  • Official homepage or docs must exist.
  • There must be a practical adoption path such as install, docs, or a clear product homepage.
  • Commercial products are allowed if their canonical docs are strong enough.
